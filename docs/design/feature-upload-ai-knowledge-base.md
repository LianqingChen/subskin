## 1. 风险应对方案

### 1.1 AI模型开发风险应对方案

#### 风险描述
开发医疗级AI诊断模型面临以下挑战：
- 医疗数据获取困难且成本高
- 模型准确性要求高（误诊风险）
- 监管合规要求（FDA/NMPA）
- 专业医生标注资源稀缺

#### 应对策略

| 策略 | 具体措施 | 实施时间 | 负责人 | 成功指标 |
|------|----------|----------|--------|----------|
| **阶段化开发** | Phase 1 (MVP): 使用预训练模型（OpenAI Vision API）进行功能验证；Phase 2: 收集用户上传的脱敏数据；Phase 3: 基于自研数据微调模型 | 立即启动 | AI工程师 | MVP 1个月内上线 |
| **多模型融合** | 集成多个预训练模型（GPT-4o Vision, Claude 3.5 Sonnet, Gemini 1.5 Pro），通过投票机制提升置信度 | Phase 1 | AI工程师 | 集成完成度100% |
| **医生审核机制** | 所有AI诊断结果需经过专业医生审核通过后才能展示给用户；建立审核队列和工作流 | Phase 1 | 产品+医生 | 审核SLA < 24小时 |
| **数据脱敏系统** | 实现自动面部马赛克、隐私信息抹除；元数据清理 | Phase 1 | 后端工程师 | 脱敏准确率 > 99% |
| **渐进式披露** | 初期仅提供疑似诊断而非确诊结果；添加显著免责声明；不提供治疗建议 | Phase 1 | 产品经理 | 法律审核通过 |
| **A/B测试框架** | 对不同模型和提示词策略进行A/B测试，持续优化准确率 | Phase 2 | 数据科学家 | 准确率提升 > 10% |
| **合规准备** | 提前准备FDA/NMPA申报材料；建立完整的模型文档和验证数据集 | Phase 2 | 法务+AI | 合规文档完整 |

#### 技术选型与成本估算

| 组件 | 选型 | 月成本 | 备注 |
|------|------|--------|------|
| 图像识别API | OpenAI GPT-4o Vision | $0.03/图 | 高准确性 |
| 备用API | Claude 3.5 Sonnet | $0.015/图 | 集成中 |
| 数据存储 | PostgreSQL + S3 | $50/月 | 100GB存储 |
| 医生审核系统 | 自研 | $0 | 人力成本 |

**Phase 1总成本**: ~$300/月（含1000次API调用）

#### 时间节点

Week 1-2:  集成OpenAI Vision API
Week 3-4:  实现数据脱敏和隐私保护
Week 5-6:  搭建医生审核工作流
Week 7-8:  MVP测试和优化
Week 9:    正式上线

### 1.2 对象存储配置风险应对方案

#### 风险描述
用户医疗照片存储面临的安全和合规挑战：
- 医疗数据隐私泄露风险
- 大文件上传性能瓶颈
- 存储成本失控
- 地域合规要求（数据本地化）

#### 应对策略

| 策略 | 具体措施 | 实施时间 | 负责人 | 成功指标 |
|------|----------|----------|--------|----------|
| **加密存储** | 客户端AES-256加密后上传；服务端不解密；密钥使用KMS管理 | 立即启动 | 后端工程师 | 加密测试通过 |
| **多地域部署** | 主存储：AWS S3 (弗吉尼亚) 备份；备份：阿里云OSS (华东) | Phase 1 | 运维工程师 | RTO < 1小时 |
| **生命周期管理** | 自动分级存储：热存储30天 → 冷存储1年 → 归档存储5年 → 删除 | Phase 1 | 运维工程师 | 成本降低60% |
| **访问控制** | IAM角色最小权限原则；每个用户独立S3路径；临时签名URL（15分钟有效） | Phase 1 | 后端工程师 | 访问审计100% |
| **内容审查** | 集成AWS Rekognition检测敏感内容；自动拒绝非医学图片 | Phase 1 | 后端工程师 | 过滤准确率 > 95% |
| **CDN加速** | 使用CloudFront/CDN加速图片加载；仅对脱敏后的图片启用CDN | Phase 2 | 运维工程师 | 加载时间 < 2秒 |
| **成本监控** | 设置CloudWatch告警：存储容量、API调用量、带宽；每日成本报告 | Phase 1 | 运维工程师 | 成本可视化100% |
| **GDPR/CCPA合规** | 实现数据删除权API；30天内自动删除被标记的图片 | Phase 1 | 后端工程师 | 删除测试通过 |

#### 存储架构设计

原图（加密）:
  S3 Bucket: subskin-private-raw/
  访问控制: 客户端加密密钥，服务端无权访问
  生命周期: 永久保存（除非用户删除）

脱敏图（可公开）:
  S3 Bucket: subskin-public-processed/
  处理: 面部马赛克、水印
  生命周期: 30天热存储 → 1年冷存储

缩略图:
  S3: subskin-thumbnails/
  用途: 列表展示
  生命周期: 7天自动清理

用户目录结构:
  subskin-private-private-raw/user_{user_id}/{timestamp}_{uuid}.enc

#### 成本优化方案

| 优化项 | 措施 | 预期节省 |
|--------|------|----------|
| 图片压缩 | WebP格式，质量85% | 40%存储 |
| 智能缩略图 | 仅生成款需尺寸（1x, 2x） | 30%带宽 |
| 批量上传 | 使用S3 Multipart Upload（>100MB） | 提升大文件速度 |
| 预签名URL | 减少75%的API开销 | 30%请求成本 |
| 冷存储 | 不常访问图片移至Glacier | 50%长期存储成本 |

**估算成本（1000用户，每用户10张图）**:
- 热存储: $10/月
- 冷存储: $5/月
- 请求成本: $20/月
- **总成本**: $35/月

## 2. 上传照片功能需求

### 2.1 用户故事

| 优先级 | 用户故事 | 验收标准 |
|--------|----------|----------|
| P0 | 作为白癜风患者，我能够上传患处照片，以便追踪病情变化 | 支持JPG/PNG/WEBP格式；单文件最大10MB；最多同时上传5张；上传进度显示 |
| P0 | 作为用户，我能够查看我之前上传的所有照片，并按日期筛选 | 照片列表按时间倒序；支持日期范围筛选；每张图显示缩略图+日期 |
| P0 | 作为用户，我能够删除不再需要的照片，保护隐私 | 删除需二次确认；删除后从列表移除；原图和缩略图都删除 |
| P1 | 作为用户，我能够为上传的照片添加备注（如"开始治疗后"） | 备注长度限制100字；备注可编辑；支持预设标签 |
| P1 | 作为用户，我能够在照片上标记患处位置（绘制区域） | 支持画圈标记；标记保存为坐标数据；支持多个标记点 |
| P2 | 作为医生，我能够查看患者上传的照片序列，用于诊断 | 按患者ID查看；时间线视图；对比功能（并排显示） |
| P2 | 作为用户，我能够下载原始照片（加密版） | 需输入密码；下载为加密文件；30分钟内有效 |

### 2.2 功能流程图

#### 主要流程：上传照片

用户操作:
  1. 点击"上传照片"按钮
  2. 选择本地图片文件
  3. (可选) 添加备注
  4. (可选) 标记患处位置
  5. 点击"确认上传"

前端处理:
  6. 验证文件格式和大小
  7. 生成缩略图（客户端）
  8. 客户端AES-256加密原图
  9. 计算文件哈希（用于去重）
  10. 显示上传进度

后端处理:
  11. 接收加密文件和元数据
  12. 生成唯一文件名
  13. 上传至S3（多部分上传，大文件）
  14. 保存元数据到PostgreSQL
  15. 创建脱敏任务（异步）
  16. 返回成功+文件ID

异步任务:
  17. 调用AI服务进行面部检测
  18. 生成马赛克脱敏图
  19. 生成缩略图（服务端）
  20. 更新数据库状态

用户反馈:
  21. 显示"上传成功"
  22. 添加到照片列表
  23. 自动触发"AI分析"功能

### 2.3 边界规则

#### 文件规则
| 规则项 | 限制 | 错误提示 |
|--------|------|----------|
| 支持格式 | JPG, PNG, WEBP | "仅支持JPG、PNG、WEBP格式" |
| 文件大小 | 最大10MB | "文件大小不能超过10MB" |
| 最小尺寸 | 100x100 px | "图片尺寸过小，建议至少100x100像素" |
| 最大尺寸 | 10000x10000 px | "图片尺寸过大，建议压缩后上传" |
| 单次上传 | 最多5张 | "单次最多上传5张图片" |
| 总上传量 | 每用户最多500张 | "已达到最大上传数量，请删除旧照片" |

#### 数据安全规则
| 规则项 | 实现 |
|--------|------|
| 原图加密 | 客户端AES-256-GCM加密，密钥由用户密码派生 |
| 传输加密 | 强制HTTPS，TLS 1.3 |
| 存储加密 | 服务端S3默认加密（SSE-S3） |
| 元数据脱敏 | EXIF数据自动清理 |
| 保留期 | 用户注销后保留30天，然后自动删除 |

## 3. AI知识库功能需求

### 3.1 用户故事

| 优先级 | 用户故事 | 验收标准 |
|--------|----------|----------|
| P0 | 作为患者，我上传照片后，系统能够自动识别疑似白癜风类型（寻常型、节段型等） | 分析时间< 30秒；显示置信度；提供3种可能结果排序 |
| P0 | 作为患者，我能够获得患处面积的估算（如"约占体表面积3%"） | 面积估算误差< 20%；提供可视化标记 |
| P0 | 作为患者，系统能够提供与症状相关的医学科普文章推荐 | 推荐文章< 5篇；相关性评分> 0.7；中文内容 |
| P1 | 作为患者，我能够查看分析历史，对比病情变化趋势 | 时间线图表；面积变化曲线；类型演变提示 |
| P1 | 作为患者，我能够向AI提出相关问题并获得医学参考答案 | 对话式交互；引用PubMed文献；明确"不构成医疗建议" |
| P2 | 作为医生，我能够查看AI的详细分析报告，辅助诊断 | 置信度热力图；特征提取结果；推荐治疗方案文献 |
| P2 | 作为研究者，我能够导出脱敏后的分析数据集（需审批） | CSV格式；包含元数据；已完全脱敏 |

### 3.2 功能流程图

#### 主要流程：AI图片分析

阶段1: 图像预处理
  ├─ 解密图片（使用用户密钥）
  ├─ 调整尺寸（适配模型输入）
  ├─ 质量增强（对比度、降噪）
  └─ 提取EXIF元数据

阶段2: 多模型推理（并行）
  ├─ 模型A: GPT-4o Vision → 疾病类型分类
  ├─ 模型B: Claude 3.5 Sonnet → 特征描述
  ├─ 模型C: Gemini 1.5 Pro → 面积估算
  └─ 模型D: 自研分割模型 → 患处分割

阶段3: 结果融合
  ├─ 投票机制（疾病类型）
  ├─ 置信度加权（特征）
  ├─ 几何计算（面积）
  └─ 生成热力图

阶段4: 知识库匹配
  ├─ 提取关键词（症状、类型、部位）
  ├─ 搜索本地知识库（向量化匹配）
  ├─ 检索PubMed文献（API）
  └─ 生成推荐列表

阶段5: 报告生成
  ├─ 生成诊断摘要
  ├─ 添加文献引用
  ├─ 生成可视化（热力图、对比图）
  └─ 标记"需医生审核"

### 3.3 边界规则

#### 分析规则
| 规则项 | 限制 | 处理方式 |
|--------|------|----------|
| 分析频率 | 每张图仅分析一次 | 重复请求返回缓存结果 |
| 分析时间 | 超时120秒 | 显示"分析中，稍后查看"，后台继续处理 |
| 置信度阈值 | < 60% | 标记"不确定"，建议医生诊断 |
| 图片质量 | 模糊/光线差 | 提示"图片质量不佳，建议重新拍摄" |

#### 知识库规则
| 规则项 | 限制 |
|--------|------|
| 推荐数量 | 每次最多返回5篇文章 |
| 内容来源 | PubMed + 本地知识库 |
| 语言要求 | 优先中文内容 |
| 时效性 | 优先推荐近5年文献 |
