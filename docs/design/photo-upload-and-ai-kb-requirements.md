## 文档概述

本文档详细定义了SubSkin平台的两个核心新功能：
1. **用户照片上传功能** - 允许白癜风患者上传病情照片，记录治疗过程
2. **AI知识库智能问答功能** - 基于上传照片进行AI分析，并关联相关知识库推荐

### 目标用户
- **主要用户**: 白癜风患者（记录病情、跟踪治疗进度）
- **次要用户**: 皮肤科医生（未来功能，查看患者照片、提供诊断建议）
- **管理员**: 系统运维、数据管理

### 业务价值
- **患者价值**: 可视化记录病情变化，获得个性化治疗建议
- **平台价值**: 积累真实世界数据，提升AI模型准确性，增强用户粘性
- **医疗价值**: 为远程医疗提供数据支持，辅助医生诊断

## 功能一：照片上传功能

### 1.1 功能概述

允许注册用户上传白癜风相关照片，系统自动生成缩略图、存储元数据，并支持按时间线查看、筛选、删除等操作。

### 1.2 用户故事

**US-1: 作为患者，我想上传我的病情照片，以便记录治疗过程中的变化**

- **验收标准**:
  - [ ] 支持JPEG、PNG、WebP格式
  - [ ] 单张照片大小限制为10MB以内
  - [ ] 上传成功后显示原图和缩略图
  - [ ] 自动提取照片的拍摄时间（EXIF）
  - [ ] 上传失败时显示友好的错误提示

**US-2: 作为患者，我想为上传的照片添加元数据（身体部位、治疗阶段、备注），以便更好地分类管理**

- **验收标准**:
  - [ ] 支持选择身体部位：脸、颈、手、躯干、腿、脚、其他
  - [ ] 支持选择治疗阶段：未治疗、治疗初期、治疗中期、稳定期、复发期
  - [ ] 支持添加文字备注（最多500字）
  - [ ] 元数据为必填项时必须验证

**US-3: 作为患者，我想按时间线查看我的所有照片，以便直观地了解病情变化趋势**

- **验收标准**:
  - [ ] 照片按拍摄时间倒序排列
  - [ ] 支持分页加载（每页20张）
  - [ ] 显示日期分隔线
  - [ ] 鼠标悬停显示照片元数据

**US-4: 作为患者，我想按身体部位或治疗阶段筛选照片，以便针对性地查看特定类型的照片**

- **验收标准**:
  - [ ] 支持按身体部位筛选（多选）
  - [ ] 支持按治疗阶段筛选（单选）
  - [ ] 支持按日期范围筛选
  - [ ] 筛选结果实时更新

**US-5: 作为患者，我想删除不需要的照片，以便保持照片库整洁**

- **验收标准**:
  - [ ] 删除前二次确认
  - [ ] 删除后从列表中移除
  - [ ] 删除操作可撤销（软删除，7天内可恢复）
  - [ ] 删除对象存储中的文件（延迟7天）

**US-6: 作为患者，我想查看照片的详细信息，以便了解完整的元数据和分析结果**

- **验收标准**:
  - [ ] 点击照片打开详情弹窗
  - [ ] 显示原图（支持缩放）
  - [ ] 显示所有元数据
  - [ ] 显示AI分析结果（如有）
  - [ ] 显示医生备注（如有，Phase 2）

## 功能二：AI知识库智能分析功能

### 2.1 功能概述

基于用户上传的照片，使用AI模型进行白癜风诊断分析，并结合知识库推荐相关的治疗指南、研究论文等，为患者提供个性化的医疗建议。

### 2.2 用户故事

**US-7: 作为患者，我想让AI分析我的照片，以便获得初步的病情评估**

- **验收标准**:
  - [ ] 分析结果包含白斑类型、严重程度、分布描述
  - [ ] 显示AI模型的置信度
  - [ ] 分析耗时 < 10秒
  - [ ] 分析失败时显示错误提示
  - [ ] 支持重新分析

**US-8: 作为患者，我想看到基于我病情的相关知识推荐，以便了解对应的治疗方案**

- **验收标准**:
  - [ ] 推荐内容包括临床指南、研究论文、治疗方案
  - [ ] 推荐内容按相关性排序
  - [ ] 显示推荐内容的高亮摘要
  - [ ] 点击推荐跳转到对应文档

**US-9: 作为患者，我想获得个性化的治疗建议，以便指导日常护理**

- **验收标准**:
  - [ ] 建议内容基于AI分析结果
  - [ ] 建议内容包含用药建议、生活方式建议、注意事项
  - [ ] 建议内容明确标注"仅供参考，不构成医疗建议"
  - [ ] 支持收藏建议内容

**US-10: 作为患者，我想对AI分析结果提供反馈，以便帮助改进AI模型**

- **验收标准**:
  - [ ] 支持评分（1-5星）
  - [ ] 支持文字反馈
  - [ ] 反馈提交成功后显示感谢提示

**US-11: 作为患者，我想查看历史分析记录，以便跟踪病情变化**

- **验收标准**:
  - [ ] 按时间显示所有分析记录
  - [ ] 显示每次分析的关键指标变化
  - [ ] 支持对比不同时间的分析结果

## 边界规则

### 照片上传边界规则

#### B-1: 文件格式限制
- **规则**: 仅允许上传JPEG、PNG、WebP格式
- **验证位置**: 前端+后端双重验证
- **违规处理**: 返回400错误，提示"仅支持JPG、PNG、WebP格式"
- **MIME类型映射**:
  - JPEG: `image/jpeg`, `image/jpg`
  - PNG: `image/png`
  - WebP: `image/webp`

#### B-2: 文件大小限制
- **规则**: 单张照片最大10MB
- **验证位置**: 前端+后端双重验证
- **违规处理**: 返回413 Payload Too Large
- **前端提示**: "文件大小不能超过10MB"

#### B-3: 上传频率限制
- **规则**: 每用户每小时最多上传10张照片
- **实现方式**: Redis计数器 + 滑动窗口算法
- **违规处理**: 返回429 Too Many Requests
- **错误提示**: "上传次数过多，请1小时后再试"

#### B-4: 重复上传检测（可选）
- **规则**: 基于SHA-256哈希值检测重复
- **实现方式**: 计算文件哈希，查询数据库
- **重复处理**: 显示警告"已存在相同照片，是否继续上传？"
- **用户选择**: 继续上传 / 取消

#### B-5: 元数据验证
- **必填字段**: `body_part`
- **选填字段**: `capture_date`, `treatment_stage`, `description`
- **数据约束**:
  - `body_part`: 枚举值（脸、颈、手、躯干、腿、脚、其他）
  - `treatment_stage`: 枚举值（未治疗、治疗初期、治疗中期、稳定期、复发期）
  - `description`: 1-500字符
- **违规处理**: 返回400 Bad Request + 具体字段错误

#### B-6: 存储降级规则
- **主存储**: 阿里云OSS / AWS S3
- **降级触发**: 连接超时、认证失败、配额不足
- **降级目标**: 本地文件系统
- **降级行为**:
  - 自动切换到本地存储
  - 记录降级日志
  - 前端显示警告："云存储暂时不可用，使用本地存储"

### AI分析边界规则

#### B-7: 分析结果缓存
- **规则**: 相同照片不重复分析
- **缓存键**: `photo_id + model_version`
- **缓存策略**: 永久缓存（除非用户删除或重新分析）
- **缓存位置**: 数据库photo_analyses表
- **过期时间**: 无（跟随照片生命周期）

#### B-8: 分析频率限制
- **规则**: 每用户每小时最多分析5次
- **实现方式**: Redis计数器
- **违规处理**: 返回429 Too Many Requests
- **错误提示**: "分析次数过多，请1小时后再试"

#### B-9: AI服务降级
- **触发条件**: 
  - 模型加载失败
  - 推理超时（>30秒）
  - 推理异常（CUDA OOM等）
- **降级目标**: Mock AI服务
- **降级行为**:
  - 记录错误日志
  - 返回Mock结果
  - 标记`degraded=true`
  - 前端显示警告："AI服务暂时不可用，使用模拟结果"

#### B-10: 分析超时保护
- **超时时间**: 30秒
- **实现方式**: 超时装饰器 / 后台任务
- **超时处理**:
  - 中断分析任务
  - 返回错误 "分析超时，请稍后重试"
  - 前端显示重试按钮

#### B-11: 置信度阈值
- **Mock置信度**: 65%-95%随机
- **真实模型置信度**:
  - < 0.5: 提示"结果可能不准确，建议咨询医生"
  - 0.5-0.7: 正常显示
  - > 0.7: 高质量结果
- **置信度展示**: 显示百分比 + 说明

#### B-12: 知识库检索规则
- **检索限制**: 每次返回Top-5推荐
- **相关性阈值**: relevance_score > 0.3
- **过滤规则**:
  - 仅返回公开文档
  - 排除已删除文档
  - 排除草稿状态
- **分类限制**: 仅检索[治疗, 指南, 研究]相关文档

### 照片管理边界规则

#### B-13: 分页规则
- **默认页大小**: 20张/页
- **最大页大小**: 100张/页
- **页码范围**: 1 ≤ page ≤ 100
- **排序规则**: 默认按capture_date DESC倒序
- **总页数计算**: `ceil(total_count / page_size)`

#### B-14: 筛选规则
- **身体部位筛选**: 支持多选（OR逻辑）
- **治疗阶段筛选**: 支持单选
- **日期范围筛选**: 支持起止日期
- **组合逻辑**: 多条件AND组合
- **结果限制**: 最多返回1000条

#### B-15: 删除恢复规则
- **软删除**: is_deleted=true, deleted_at=NOW()
- **恢复窗口**: 删除后7天内可恢复
- **过期判断**: deleted_at < NOW() - 7 days
- **存储清理**: 延迟7天删除对象存储文件
- **硬删除**: 可选功能，需管理员权限

#### B-16: 数据所有权验证
- **规则**: 严格验证photo.user_id == current_user.id
- **查询过滤**: 所有查询自动加上user_id过滤
- **跨用户访问**: 禁止（返回403 Forbidden）
- **管理员权限**: 可查看所有照片（审计用）

---

### 反馈系统边界规则

#### B-17: 反馈评分规则
- **评分范围**: 1-5星（整数）
- **必填字段**: rating
- **选填字段**: comment, helpful
- **字符限制**: comment最多500字符
- **重复反馈**: 允许同一照片多次反馈

#### B-18: 反馈统计规则
- **实时更新**: 每次反馈后重新计算平均分
- **统计维度**:
  - 总反馈数
  - 平均评分
  - 有帮助占比
  - 按评分分布（1-5星各多少）
- **展示位置**: AI分析结果页、管理后台
