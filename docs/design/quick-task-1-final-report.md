## 二、风险应对方案总结

### 风险一：AI模型开发风险

**应对策略**: 三阶段渐进式交付

| 阶段 | 时间 | 目标 | 风险降低 |
|------|------|------|----------|
| **阶段1: Mock服务** | Day 1 | 快速验证业务逻辑和用户体验 | 60% |
| **阶段2: 轻量级模型** | Day 2 | 集成预训练模型，提供基础诊断能力 | 80% |
| **阶段3: 专业模型** | Phase 2 | 训练专业级模型，达到生产要求 | 95% |

**关键技术方案**:
- Mock智能返回：基于照片元数据生成"合理"分析结果
- 轻量级模型：UNet/SAM分割 + ResNet-50分类
- 降级策略：真实模型失败自动降级到Mock服务

---

### 风险二：对象存储配置风险

**应对策略**: 存储抽象层 + 多级降级

**降级链路**:
```
阿里云OSS / AWS S3 (主存储)
    ↓ (失败)
本地文件系统 (降级备选)
    ↓ (失败)
内存存储 (最后防线)
    ↓ (失败)
返回错误
```

**关键方案**:
- 统一存储接口（StorageBackend抽象基类）
- 自动降级逻辑（遍历后端列表）
- 健康检查端点（实时监控各后端状态）
- 配置化切换（环境变量控制）

## 三、详细需求文档要点

### 功能一：照片上传功能（US-1 至 US-6）

- **文件格式**: JPEG、PNG、WebP
- **文件大小**: ≤10MB
- **核心能力**:

  - ✅ 病情阶段判断（进展期/稳定期）
  - ✅ 知识库推荐（临床指南、研究论文）
  - ✅ 个性化建议（用药、生活方式、注意事项）

- **管理员功能**:
  - ✅ 上传新文档（PDF、Word、Markdown）
  - ✅ 手动触发向量化
  - ✅ 批量处理、进度监控

## 四、边界规则摘要

### 照片上传边界规则

| 规则 | 具体要求 | 违规处理 |
|------|---------|---------|
| 文件格式 | JPEG、PNG、WebP | 400错误 |
| 文件大小 | ≤10MB | 413错误 |
| 上传频率 | 10次/小时 | 429 Too Many Requests |
| 重复上传 | SHA-256检测 | 警告提示 |
| 存储降级 | 自动切换 | 前端警告 |

### AI分析边界规则

| 规则 | 实现方式 | 说明 |
|------|---------|------|
| 结果缓存 | photo_id + model_version | 永久缓存 |
| 分析频率 | 5次/小时 | Redis限流 |
| AI服务降级 | 自动降级到Mock | 异常处理 |
| 超时保护 | 30秒超时 | 超时装饰器 |
| 置信度阈值 | <0.5提示不准确 | 策略判断 |
| 知识库检索 | 混合检索 | Top-5推荐 |

## 五、功能流程图要点

### 照片上传主流程
```
选择文件 → 验证格式/大小 → 上传到对象存储 → 
提取EXIF → 生成缩略图 → 计算SHA-256 → 保存到数据库 → 
[可选]触发AI分析
```

### AI分析主流程
```
触发分析 → 检查缓存 → 调用AI服务（Mock/真实模型） → 
降级保护 → 保存结果 → 触发知识库检索 → 
生成个性化建议 → 返回完整结果
```

### 存储降级流程
```
主存储失败 → 检查本地存储 → 使用本地文件系统 → 
标记降级 → 返回本地URL
```

## 七、API接口设计摘要

### 照片管理接口
- `POST /api/photos` - 上传照片
（JWT认证）
- `GET /api` - 获取照片列表（支持分页、筛选）
- `GET /api/photos/{id}` - 获取照片详情
- `DELETE /api/photos/{id}` - 删除照片（软删除）
- `POST /api/photos/{id}/restore` - 恢复已删除照片

### AI分析接口
- `POST /api/photos/{id}/analyze` - 触发AI分析
- `GET /api/photos/{id}/analysis` - 获取分析结果
- `POST /api/photos/{id}/analysis/feedback` - 提交反馈（评分1-5星）

### 系统接口
- `GET /api/storage/health` - 存储服务健康检查

## 八、安全方案要点

### 认证与授权
- ✅ JWT认证（所有接口）
- ✅ 数据所有权验证（photo.user_id == current_user.id）
- ✅ 权限分级（普通用户/管理员/医生）

### 数据加密
- ✅ 传输加密：TLS 1.2+
- ✅ 存储加密：AES-256
- ✅ 对象存储：服务器端加密（SSE-S3）

### 隐私保护
- ✅ 数据最小化：仅收集必要元数据
- ✅ 数据保留：用户删除后7天清除
- ✅ 用户控制：提供数据导出、删除功能

### 访问控制
- ✅ 速率限制：上传10次/小时，分析5次/小时
- ✅ IP白名单：管理员接口
- ✅ 审计日志：所有敏感操作
